## 📝 編集可能なコンテンツ

サーバーにアップロードした`build`フォルダ内の以下のファイルを編集できます：

```
build/
└── data/
    ├── news.en.json      # ニュース（英語）
    ├── news.ja.json      # ニュース（日本語）
    ├── works.en.json     # 実績（英語）
    ├── works.ja.json     # 実績（日本語）
    ├── services.en.json  # サービス（英語）
    ├── services.ja.json  # サービス（日本語）
    ├── company.en.json   # 会社情報（英語）
    ├── company.ja.json   # 会社情報（日本語）
    ├── content.en.json   # その他コンテンツ（英語）
    └── content.ja.json   # その他コンテンツ（日本語）
```

## 🔧 編集手順

### ステップ1: ファイルをダウンロード

1. Conohaサーバーから編集したいJSONファイルをダウンロード
2. 例：`build/data/news.ja.json`

### ステップ2: テキストエディタで開く

以下のいずれかのエディタで開いてください：

- **Windows**: メモ帳、Notepad++、Visual Studio Code
- **Mac**: テキストエディット、Visual Studio Code
- **推奨**: Visual Studio Code（無料、見やすい）

### ステップ3: 内容を編集

JSONファイルの内容を編集します。詳しくは[各ファイルの説明](#各ファイルの説明)を参照。

### ステップ4: 保存

- **文字コード**: UTF-8
- **改行コード**: LF（推奨）またはCRLF

### ステップ5: サーバーにアップロード

編集したファイルを元の場所（`build/data/`）にアップロードします。

### ステップ6: 確認

ブラウザでウェブサイトを開き、変更が反映されているか確認します。
**キャッシュクリア**: Ctrl+F5（Windows）/ Cmd+Shift+R（Mac）

## 📚 各ファイルの説明

### 1. ニュース（news.ja.json / news.en.json）

```json
{
  "hero": {
    "title": "ニュース",
    "subtitle": "最新情報"
  },
  "news": [
    {
      "id": "2026-001",
      "date": "2026-04-24",
      "category": "お知らせ",
      "title": "公式ウェブサイトリニューアルのお知らせ",
      "summary": "この度、公式ウェブサイトを全面リニューアルいたしました。"
    }
  ]
}
```

#### 編集方法

**新しいニュースを追加する場合**:

```json
{
  "news": [
    {
      "id": "2026-002",
      "date": "2026-05-01",
      "category": "イベント",
      "title": "新製品発表会開催のお知らせ",
      "summary": "6月1日に新製品発表会を開催いたします。"
    },
    {
      "id": "2026-001",
      "date": "2026-04-24",
      "category": "お知らせ",
      "title": "公式ウェブサイトリニューアルのお知らせ",
      "summary": "この度、公式ウェブサイトを全面リニューアルいたしました。"
    }
  ]
}
```

**ポイント**:
- 新しいニュースは配列の**先頭**に追加
- `id`は一意の値（例: 年-連番）
- `date`は「YYYY-MM-DD」形式
- カンマ（`,`）の位置に注意

---

### 2. 実績（works.ja.json / works.en.json）

```json
{
  "hero": {
    "title": "実績",
    "subtitle": "これまでの取り組み"
  },
  "works": [
    {
      "id": "work-001",
      "title": "プロジェクト名",
      "category": "カテゴリー",
      "year": "2025",
      "description": "プロジェクトの説明文",
      "image": "/images/works/project-001.jpg",
      "tags": ["タグ1", "タグ2"]
    }
  ]
}
```

#### 編集方法

- `title`: プロジェクト名
- `category`: カテゴリー（例: Webサイト制作、システム開発）
- `year`: 実施年
- `description`: 詳細説明
- `image`: 画像パス（`/images/works/`フォルダ内）
- `tags`: 関連タグの配列

---

### 3. サービス（services.ja.json / services.en.json）

```json
{
  "hero": {
    "title": "サービス",
    "subtitle": "提供サービス"
  },
  "services": [
    {
      "id": "service-001",
      "title": "サービス名",
      "description": "サービスの説明",
      "features": [
        "特徴1",
        "特徴2",
        "特徴3"
      ]
    }
  ]
}
```

## ⚠️ 注意事項

### 必ず守ること

1. **JSON形式を崩さない**
   - `{` と `}` の対応
   - `[` と `]` の対応
   - カンマ（`,`）の位置
   - ダブルクォーテーション（`"`）で文字列を囲む

2. **文字コードはUTF-8で保存**
   - 文字化けを防ぐため

3. **バックアップを取る**
   - 編集前に必ずオリジナルファイルをコピー

4. **一度に大量の変更をしない**
   - 少しずつ編集して確認

### よくあるミス

❌ **間違い**: カンマが抜けている
```json
{
  "title": "タイトル"
  "subtitle": "サブタイトル"
}
```

✅ **正しい**:
```json
{
  "title": "タイトル",
  "subtitle": "サブタイトル"
}
```

❌ **間違い**: 最後の要素にカンマがある
```json
{
  "news": [
    {"id": "001", "title": "ニュース1"},
    {"id": "002", "title": "ニュース2"},
  ]
}
```

✅ **正しい**:
```json
{
  "news": [
    {"id": "001", "title": "ニュース1"},
    {"id": "002", "title": "ニュース2"}
  ]
}
```

## 🔍 トラブルシューティング

### サイトが表示されなくなった

**原因**: JSON形式のエラー

**解決方法**:
1. バックアップファイルで上書き
2. JSONチェッカーで確認（https://jsonlint.com/）
3. エラー箇所を修正

### 変更が反映されない

**原因**: ブラウザのキャッシュ

**解決方法**:
- **Windows**: Ctrl + F5
- **Mac**: Cmd + Shift + R
- それでもダメな場合: ブラウザのキャッシュをクリア

### 文字化けする

**原因**: 文字コードが間違っている

**解決方法**:
1. UTF-8で保存し直す
2. Visual Studio Codeなどのエディタを使用

## 📌 補足: JSONチェッカーの使い方

1. https://jsonlint.com/ にアクセス
2. 編集したJSONファイルの内容をコピー
3. 貼り付けて「Validate JSON」をクリック
4. エラーがあれば赤字で表示されます
5. エラーを修正して再度チェック
